# AI Usage Disclosure & Reflection

## Overview of AI Usage
Throughout the development of CapstoneIQ, I utilized the **Antigravity (Gemini 3.1 Pro)** AI agent acting as an autonomous pair programmer. The AI was instrumental in scaffolding the Next.js architecture, porting over React components from previous projects, and designing the integration pipeline UI that maps sentiment scores to DCF inputs.

## Specific AI Contributions
1. **Architecture & Scaffolding**: The AI agent successfully executed the `npx create-next-app` command and configured Tailwind CSS (v4) with custom Google Fonts (Syne, JetBrains Mono).
2. **Logic Porting**: The AI seamlessly ported the Vanilla React logic from `dcf-valuation.jsx` into a modular, type-safe Next.js environment (`src/lib/valuationEngine.ts`).
3. **Integration Design**: When prompted to ensure the project met the "Excellent" tier of the grading rubric, the AI designed the 3-step "Cross-Component Integration Engine" UI. This explicitly exposes the underlying math, making it easy to explain the data flow during the video presentation.
4. **Data Generation**: The application utilizes the **Anthropic API (Claude)** dynamically. Rather than building a backend scraper, the app prompts Claude to act as a financial data API, extracting base financial metrics and current market sentiment in a single JSON payload.

## Limitations & Risks of the AI System
1. **Hallucination Risk**: Because the financial and sentiment data is generated by an LLM (Claude) via prompt engineering rather than pulled from a verified Bloomberg or SEC database, the data is subject to hallucination. It is strictly for educational demonstration of the *pipeline*, not for actual financial advisory.
2. **Deterministic vs Probabilistic**: The DCF engine is purely deterministic (math-based), while the sentiment engine is probabilistic (LLM-based). Combining the two means the final Intrinsic Value carries the uncertainty and bias inherent in the LLM's sentiment analysis.

## Reflection
Using an agentic AI assistant allowed me to rapidly build a highly polished, professional-grade interface that effectively visualizes complex data flows. The hardest part was ensuring the integration was explicit—rather than just "magic" happening behind the scenes, the AI helped me build a UI that *shows the work*, which is exactly what a professional analyst would require for due diligence.
